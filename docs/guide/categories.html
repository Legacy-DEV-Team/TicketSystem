Ticket Categories

Complete guide to creating and managing ticket categories

Categories Overview

Ticket categories allow you to organize different types of support requests. This guide covers:

  • Creating and configuring categories
  • Setting up category-specific permissions
  • Customizing category appearance
  • Managing category workflows
  • Advanced category features

Creating Categories

1. Access Category Management

  1. Log into your dashboard
  2. Select your Discord server
  3. Navigate to "Ticket Categories"
  4. Click "Create Category"

2. Basic Category Settings

General Information

  • Name: Descriptive name (e.g., "General Support")
  • Description: Brief description of the category
  • Emoji: Visual identifier (e.g., üé´, üêõ, üí∞)
  • Color: Category color for visual distinction

Channel Configuration

  • Channel: Discord channel where tickets will be created
  • Button Text: Text displayed on the ticket button
  • Button Style: Primary, Secondary, Success, Danger

3. Permission Settings

Access Control

  • Required Roles: Roles needed to create tickets
  • Blocked Roles: Roles prevented from creating tickets
  • Staff Roles: Roles that can manage tickets

Visibility

  • Public: Visible to all users
  • Private: Only visible to staff
  • Hidden: Completely hidden from users

Advanced Category Settings

1. Auto-Close Configuration

Inactivity Settings

  • Auto-Close Enabled: Enable/disable automatic closing
  • Inactivity Time: Hours before auto-close (default: 72)
  • Warning Time: Hours before warning (default: 24)
  • Warning Message: Custom warning message

2. Notification Settings

Staff Notifications

  • Staff Notifications: Notify staff of new tickets
  • Log Channel: Channel for ticket logs
  • Transcript Channel: Channel for transcripts
  • Mention Staff: Mention staff roles in notifications

3. Custom Fields

Add custom fields to ticket creation:

  • Text Fields: Short text inputs
  • Text Areas: Long text inputs
  • Dropdowns: Predefined options
  • Checkboxes: Multiple choice options

4. Workflow Automation

  • Auto-Assign: Automatically assign tickets to staff
  • Escalation Rules: Escalate tickets based on criteria
  • Tagging: Auto-tag tickets based on content
  • Priority Assignment: Set priority based on user role

Managing Categories

1. Editing Categories

  1. Go to "Ticket Categories" in the dashboard
  2. Click the edit icon next to the category
  3. Modify the settings as needed
  4. Click "Save Changes"

2. Reordering Categories

  • Drag and drop categories to reorder
  • Order affects button placement in Discord
  • Most important categories should be first
  • Changes apply immediately

3. Disabling Categories

  • Temporarily disable categories without deleting
  • Disabled categories don't show buttons
  • Existing tickets remain active
  • Can be re-enabled at any time

4. Deleting Categories

Warning: Deleting a category will remove all associated buttons and settings. Existing tickets will remain but new tickets cannot be created.

Best Practices

1. Category Design

  • Use clear, descriptive names
  • Keep categories focused and specific
  • Use appropriate emojis and colors
  • Limit the number of categories (5-10 recommended)

2. Permission Strategy

  • Start with open categories and add restrictions as needed
  • Use role-based access for specialized categories
  • Regularly review and update permissions
  • Document permission requirements

3. Workflow Optimization

  • Set appropriate auto-close times
  • Configure escalation rules for urgent issues
  • Use custom fields to gather necessary information
  • Set up proper notification channels

4. Maintenance

  • Regularly review category performance
  • Update categories based on usage patterns
  • Clean up unused categories
  • Monitor and optimize workflows

Troubleshooting

Common Issues

Buttons Not Appearing

  • Check if category is enabled
  • Verify bot permissions in channel
  • Check category order and visibility
  • Refresh Discord client

Permission Errors

  • Verify required roles are set correctly
  • Check if user has required roles
  • Review blocked roles configuration
  • Test with different user accounts

Auto-Close Not Working

  • Check auto-close is enabled
  • Verify inactivity time settings
  • Check bot permissions
  • Review warning message configuration

Notifications Not Sending

  • Verify notification channels exist
  • Check bot permissions in channels
  • Review staff role assignments
  • Test notification settings
